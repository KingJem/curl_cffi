# curl_cffi Source Build Guide

This fork builds `curl_cffi` against the `libcurl-impersonate` release assets
published in `KingJem/curl-impersonate`.

The default build path is intentionally release-based: it does not prefer a
locally compiled `libcurl-impersonate.a`, and `libs.json` should not contain a
`local_libdir` override.

## Current Versions

| Component | Version | Source |
|---|---:|---|
| curl_cffi | 0.15.3 | `KingJem/curl_cffi` |
| curl-impersonate | 1.5.3 | `KingJem/curl-impersonate` GitHub Release |
| curl | 8.15.0 | patched headers generated by `make preprocess` |

## Install From GitHub

Use this command to simulate a clean machine installing from your fork:

```bash
python3 -m venv /tmp/curl-cffi-install-test
/tmp/curl-cffi-install-test/bin/python -m pip install -U pip wheel setuptools
/tmp/curl-cffi-install-test/bin/python -m pip install --no-cache-dir --force-reinstall \
  'curl_cffi @ git+https://github.com/KingJem/curl_cffi.git@main'
```

During the build, `scripts/build.py` should print a download URL like:

```text
https://github.com/KingJem/curl-impersonate/releases/download/v1.5.3/libcurl-impersonate-v1.5.3.<arch>-<platform>.tar.gz
```

For macOS arm64, the expected asset is:

```text
libcurl-impersonate-v1.5.3.arm64-macos.tar.gz
```

## Local Wheel Build

From this repository:

```bash
python3 -m pip install -U pip wheel setuptools build
make preprocess
python3 -m build --wheel
```

The wheel is written to `dist/`.

`make preprocess` downloads the `curl-impersonate` source tag and copies patched
curl headers into `include/curl/`. The actual `libcurl-impersonate` library used
by the CFFI extension still comes from the GitHub Release asset configured in
`scripts/build.py`.

## Verification

Run this outside the repository directory so Python imports the installed wheel,
not the local working tree:

```bash
cd /tmp
/tmp/curl-cffi-install-test/bin/python - <<'PY'
import curl_cffi
from curl_cffi import Curl

print("curl_cffi version:", curl_cffi.__version__)
print("curl build version:", curl_cffi.__curl_version__)

c = Curl()
try:
    print("curl version:", c.version().decode())
finally:
    c.close()
PY
```

Expected output contains:

```text
curl_cffi version: 0.15.3
libcurl/8.15.0-IMPERSONATE
ngtcp2
nghttp3
```

## Profile Smoke Test

```bash
python3 - <<'PY'
from curl_cffi import requests

url = "https://tls.browserleaks.com/json"

for profile in ["brave136_mac", "brave145_mac", "tor1508"]:
    response = requests.get(url, impersonate=profile, timeout=20)
    response.raise_for_status()
    data = response.json()
    print(profile, data.get("ja4"), data.get("user_agent"))
PY
```

Network fingerprint endpoints can be flaky, so this is a smoke test rather than
a deterministic unit test.

## Important Files

| Path | Purpose |
|---|---|
| `scripts/build.py` | Selects the platform asset and downloads `libcurl-impersonate` from GitHub Releases. |
| `libs.json` | Maps Python platform info to the expected release asset suffix and link type. |
| `Makefile` | Generates patched curl headers and builds wheels locally. |
| `curl_cffi/requests/custom_profiles.py` | Custom Brave, Tor, OkHttp, and mobile-oriented profile definitions. |
| `curl_cffi/requests/impersonate.py` | Public profile literals and fingerprint dataclasses. |
| `examples/brave_method1.py` | Brave-style request headers layered on Chrome impersonation. |
| `examples/custom_fp_method3.py` | Fully custom JA3/Akamai/extra_fp request examples. |

## Notes

- Do not add `local_libdir` back to `libs.json` for normal builds.
- If a `curl-impersonate` release asset is missing, publish the asset first in
  `KingJem/curl-impersonate`, then rebuild `curl_cffi`.
- macOS arm64 uses static linking against `libcurl-impersonate.a`, so the built
  wheel does not need a separate `.dylib` next to `_wrapper.abi3.so`.
